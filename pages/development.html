node.js

All of my node projects use Bob to build. This involves running lint, complexity check, unit tests, tests coverage check, integration tests, and API documentation.

Install Bob (once-off, unless you want to upgrade):

npm install -g bob

Do your development, and don't forget to add test(s). Each code change should involve either a new or modified test.

Before you commit, build the project from its root directory (same level as package.json):

bob build

If everything passes, then you can commit and/or push.

Less than 100% coverage is currently allowed, but please aim for 100% coverage. I understand that it's painful to cover all error cases but it will pay off, trust me.

There's currently no minimum complexity threshold. But please aim for greater than 65 on average maintainability.

After you push, Travis CI will then kick in to make sure that your fork builds.
